Career Jump 1.6.4 restores the stable scan core while keeping the 1.6 user experience improvements.

This release rolls the underlying Workday and inventory execution path back toward the proven 1.5.2 behavior after regressions in the 1.6 core flow caused scans to build inventory without actually saving results, refreshing the dashboard, or sending notifications. The application now keeps the improved 1.6 interface and scan controls while using a simpler, more reliable scan pipeline again.

What’s improved

- Restored the simpler Workday scan execution path so boards that were stable in 1.5.x are handled the same way again.
- Removed the newer timeout and page-progress orchestration from the Workday core path, which was interfering with successful completion.
- Restored the simpler inventory finalization sequence so successful scans save inventory, update counts, and complete notifications correctly again.
- `Abort Scan` still exists in the product UI, but abandoned runs are now prevented from overwriting the results of a newer run at finalization time.
- The scan monitor no longer treats “inventory built” as a final completed state; it only announces completion after the real terminal events occur.

Why this release matters

- Fixes regressions where scans could appear complete in the UI while the dashboard, inventory, and email never actually updated.
- Reverts the most failure-prone 1.6 core changes without giving up the improved 1.6 interface, footer, configuration UX, themes, and overall product polish.
- Brings the app back closer to the reliable 1.5.x behavior you were using successfully in production.

Operational notes

- Architecture docs were updated to reflect the restored simpler scan core and the current abort/lock behavior.
- The explicit broken-link cleanup action remains available, but is no longer part of the normal `/run` lifecycle.
