# Career Jump 1.4.0

This release focuses on reliability, visibility, and operational polish. It brings the latest stable application improvements to production while keeping the core job-tracking experience familiar.

## What's improved

- Scheduled run behavior is easier to understand and troubleshoot.
- Manual run failures are easier to diagnose with clearer in-app logging.
- Company-level scan summaries remain available in the app for faster debugging when a source returns fewer jobs than expected.
- Production release notes now follow a cleaner product-style format so updates are easier to understand.

## Why this matters

- You can trust scheduled run timing more confidently.
- If a scan fails or behaves unexpectedly, there is better information available in `/logs`.
- Operational changes are easier to follow without digging through engineering-only changelogs.

## Reliability notes

- No user action is required after this update.
- Existing saved configuration and tracked job state remain intact.
- This release is intended to improve supportability and release quality rather than change your workflow.
