# Career Jump API 3.0.0

## Overview

Version 3.0.0 introduces a larger operational backend for Career Jump. This release extends the admin API surface, adds announcement and documentation support, expands company-level applied-job retrieval, and brings the registry scan platform onto dedicated queue-backed infrastructure for adapter-specific background processing.

## Highlights

- Added queue-backed registry scanning infrastructure with dedicated Workday, enterprise, and public-API SQS queues.
- Added scheduler and worker foundations for periodic registry-company scans, with queue routing by adapter class.
- Expanded admin API support for analytics, support, Stripe configuration, plan configuration, live announcements, and embedded documentation workflows.
- Added data-export utilities for registry scan state and applied-job records to support admin and operational analysis.
- Expanded OpenAPI coverage so the shipped admin and operational routes are reflected in the live API documentation surface.

## Platform and Infrastructure Changes

- Added `WorkdayQueue`, `EnterpriseQueue`, and `PublicApiQueue` to separate registry scan workloads by adapter family.
- Added `RegistrySchedulerFunction` to dispatch due companies into the appropriate queue based on adapter type and scan pool.
- Added `RegistryScanWorkerFunction` event-source mappings for all three queue classes.
- Added `RegistryOpsDashboardFunction`, dashboard alarms, metric filters, log groups, and scheduler resources for operational registry visibility.
- Updated the production API IAM permissions so runtime admin analytics and registry-backed write flows can persist their operational cache/state safely.

## API and Data Model Changes

- Broadened route coverage across admin, billing, support, applied-job, and company-level workflow endpoints.
- Expanded storage exports and registry scan-state persistence helpers used by operations and diagnostics.
- Continued enriching runtime types and storage layers to support the new queue-driven registry-scan model and broader admin workflows.
- Kept the queue-based rollout compatible with the existing production Lambda functions while introducing new scheduler and worker components.

## Operational Impact

- Registry scans can now be scheduled and routed by adapter family instead of relying on a single generic background path.
- The backend now has the infrastructure required to scale periodic scan orchestration more deliberately across Workday, enterprise ATS providers, and public API adapters.
- Admin and support operations benefit from a larger live API surface and richer documentation of shipped contracts.

## Risk and Compatibility Notes

- This release adds new AWS infrastructure, including queues, scheduler resources, worker Lambdas, alarms, metric filters, and log groups.
- Successful deployment requires the deployer IAM user to have SQS queue-creation permissions in addition to the existing Lambda and CloudFormation rights.
- Existing core API functions remain compatible with the current app shell, while new resources extend the background scan architecture rather than replacing it outright.
