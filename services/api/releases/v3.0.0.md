# v3.0.0 — 2026-04-23

## Summary

Major release marking Career Jump AWS as the primary production runtime. Includes security hardening, UI improvements, storage layer refactoring, structured logging, full GitHub migration, and a complete documentation overhaul.

## What's New

### Security
- S3 bucket is now fully private — CloudFront Origin Access Control (OAC) blocks all direct S3 URL access
- CloudFront Response Headers Policy adds HSTS (2-year), X-Frame-Options: DENY, X-Content-Type-Options, XSS protection, and Content Security Policy to every response

### Infrastructure
- EventBridge Scheduler enabled: weekday scans every 3 hours, 6am–9pm ET (Mon–Fri)
- ScanCompany Lambda timeout reduced from 480s to 180s; memory from 1024 MB to 256 MB
- CloudWatch error metric filters and alarms added for all four Lambda functions
- SNS topic + email subscription for Lambda error alerts

### UI — Available Jobs
- Posted at and Location columns added
- Table column widths normalized with `table-layout: fixed`
- Location column truncates with ellipsis; full value on hover
- Notes column repositioned after Job title
- Discard button removed from per-row actions
- Bulk action bar (mark applied / remove / clear) restored to floating bottom position
- "+ Add job" button enlarged for visibility

### UI — Applied Jobs
- Posted at column added (after Applied at)
- Column widths switched to browser auto-layout for natural sizing
- Status dropdown min-width reduced to 120px

### Storage
- `src/storage.ts` split into domain modules: `locks`, `applied`, `inventory`, `filters`, `logs`, `overrides`
- Ambiguous TypeScript barrel re-export collision fixed

### Logging
- Structured logger added at `src/lib/logger.ts`
- Observability architecture documented at `docs/architecture/observability-and-logging.md`

### Operations
- Manual job add (quick-add modal on Available Jobs)
- Email webhook URL and shared secret configurable from the app UI (Configuration tab)
- Email service falls back to env vars when no UI-configured secret is present

### Release Process
- Migrated from GitLab to GitHub (`git@github.com:dipak-bhujbal/career-jump-aws.git`)
- CI/CD pipeline removed — releases are manual `sam deploy` from developer terminal
- `docs/release-runbook.md` added as the authoritative release and deploy guide

### Documentation
- All docs updated from GitLab → GitHub and from POC → production framing
- Architecture diagrams updated: GitLab node removed, EventBridge shown as enabled
- `PROJECT_INSTRUCTIONS.md` updated with correct Lambda specs, version, and deploy process

## Validation

- `npm run check` passed
- `sam validate --lint --region us-east-1` passed
- `npm run aws:build` passed
