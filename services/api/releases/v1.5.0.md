# Career Jump 1.5.0

This release simplifies the app into a true single-workspace production setup and fixes two sources of confusing behavior in scheduled scans.

## What’s improved

- Removed runtime tenant separation from the active application flow so manual runs, scheduled runs, logs, dashboard data, and notifications all operate on the same workspace state.
- Strengthened geography filtering for Workday jobs by checking the job URL in addition to the displayed location text, which helps block non-US roles that were slipping through vague multi-location labels.
- Kept the direct Cloudflare cron schedule and cleaned up the runtime so scheduled runs and browser views stay in sync.

## Why this matters

- Scheduled runs now update the same inventory, logs, and dashboard state you see in the UI.
- Notifications should no longer drift away from the inventory because of split workspace state.
- Non-US Workday roles, including cases like Israel/Tel Aviv encoded in the job URL, are much less likely to be reported as valid new jobs.

## No action needed

Your existing workflow stays the same after this release.
