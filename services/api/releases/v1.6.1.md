Career Jump 1.6.1 restores a smoother scan completion experience.

This hotfix fixes a regression introduced in 1.6.0 where the app could finish building inventory but still look like a scan was running. The dashboard now refreshes properly when a scan finishes, the scan button returns to its idle state, and completion toasts clear the way they should.

## What’s fixed

- Scan completion now updates the dashboard and inventory reliably after the build finishes.
- The sidebar action now returns from `Abort Scan` back to `Scan` when a run is actually done.
- Completion and abort messages now close out the live scan state correctly instead of leaving the UI stuck in an in-progress state.

## Reliability notes

- This is a targeted frontend hotfix for the 1.6.0 scan-monitor regression.
- No scan rules, scheduling, or configuration behavior changed in this release.
