# Career Jump 1.3.2

This release improves scheduling visibility and makes failed manual scans much easier to diagnose without changing how your saved settings or tracked job data work.

## What's improved

- Scheduled scans now follow the intended Eastern Time window more reliably.
- The app now exposes a schedule debug endpoint so support and troubleshooting are clearer when a run is expected to wait for the next allowed window.
- Manual scan failures are now recorded explicitly in the in-app logs instead of failing without enough context.

## Why this matters

- You can trust the weekday run window more easily.
- If a manual run fails, the reason is easier to find in `/logs`.
- Troubleshooting scheduled behavior no longer depends on guesswork.

## Reliability notes

- This release keeps the original in-app `/logs` debugging flow.
- No Grafana or Worker-observability integration is part of this release.
- No user configuration changes are required.
