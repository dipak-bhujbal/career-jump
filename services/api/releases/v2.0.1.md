# Career Jump v2.0.1

Career Jump v2.0.1 tightens the AWS POC storage lifecycle and makes `/logs.html` more useful for per-company troubleshooting.

## What Changed

- Available inventory now stores only jobs that pass the current filters and are still available.
- Applying a job preserves the applied-job record, removes only the available inventory copy, and keeps the existing Applied Jobs flow intact.
- Moving an applied job to `Interview` continues to create interview-round state so it appears in Action Plan.
- Discarded fetched jobs are no longer persisted as job records; scan diagnostics keep counts and reason summaries instead.
- Broken links and jobs missing from a later scan are removed from available inventory.
- App logs and temporary per-company decision summaries expire after six hours through DynamoDB TTL.
- Trend history recycles repeated unchanged points instead of appending duplicate count snapshots.
- `/logs.html` now shows one compact row per company per run, including all major counts, failures, duration, progress, and updated-job previous/current diffs.

## Operator Notes

- Deploy this release through the protected GitLab `release-prod` pipeline from tag `v2.0.1`.
- Local production deploy is intentionally not part of this release path.
- `/api/logs?compact=false` remains available for raw short-lived progress rows during the six-hour log TTL.
