# Career Jump v1.1.0

**Released:** 2026-04-27

## Summary

Career Jump v1.1.0 is the release branch used to publish and deploy the first unified GitHub-backed production baseline. It carries forward the full multi-tenant, admin/support, SES, and US-only edge foundations from v1.0.0 while incorporating the deployment-safety fixes uncovered during the first production rollout attempts.

## Highlights

- **Release branch alignment** so the monorepo can be pushed and deployed from `release/1.1.0`.
- **Deploy-safe Cognito admin pool configuration** by disabling MFA enforcement in the initial stack template until the admin sign-in flow is hardened post-launch.
- **No product-scope rollback** of the admin, support, session, or geofencing work completed in the v1.0.0 baseline.

## Deployment-Safety Changes

- Updated the admin Cognito user pool template to avoid the `MFA optional requires SMS/phone verification` create failure during first-stack provisioning.
- Rebuilt the SAM artifacts so the generated deployment template now matches source control.
- Preserved the CloudFront-based US geofence, origin verification, and break-glass logging model introduced in the baseline release.

## Verification

- `npm run validate`
- `sam validate --lint --region us-east-1`
- `npm run aws:build`

## Deployment Notes

- Deploy this release from the `release/1.1.0` branch.
- Delete any prior `ROLLBACK_COMPLETE` `career-jump-prod` stack shell before retrying deployment.
- Provide `OriginVerifyHeaderValue` at deploy time, not in source control.
- The deployer IAM user still needs sufficient IAM role permissions for Lambda execution-role creation and tagging.

## Rollback

- Roll back by redeploying the previous stable branch or tag once the AWS account is fully verified.
- No schema-specific rollback steps are required beyond infrastructure recreation.
