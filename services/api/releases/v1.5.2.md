# Career Jump 1.5.2

This release makes scan execution safer and upgrades company filtering across the app so it is faster to narrow inventory, applied jobs, and logs without hand-typing company names.

## What’s improved

- Added a single active run lock so manual and scheduled scans no longer start in parallel and compete to overwrite the same workspace state.
- Replaced free-text company filters with Excel-style multi-select dropdowns on available jobs, applied jobs, and logs.
- Removed the redundant `US only` filter from the available-jobs page now that geography filtering is expected to happen before roles enter inventory.
- Trimmed repetitive company completion logging while keeping the useful new, updated, and discarded details in the completion record itself.

## Why this matters

- Refreshing the browser during a scan no longer lets a second `/run` start and race the first one.
- Company filtering is more precise and easier to use, especially when you want to compare a few target companies at once.
- The available-jobs filters now better reflect the product rule that non-US roles should be excluded before they reach the inventory.
- Logs stay compact without losing the details that matter for debugging a company scan.

## Reliability notes

- If a scan is already running, starting another one now returns a clear “run already in progress” response instead of launching a second concurrent scan.
- Scheduled runs use the same lock, so background scans will wait their turn instead of overlapping a manual run.
