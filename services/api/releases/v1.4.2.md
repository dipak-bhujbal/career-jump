# Career Jump 1.4.2

This release tightens the production runtime configuration and removes the last unfinished observability experiment.

## What’s improved

- Removed the remaining Grafana/Cloudflare observability export configuration from production.
- Restored a simpler logging model centered on the in-app `/logs` experience.
- Increased the Worker CPU budget for requests to 90 seconds to give long-running scans more headroom.

## Why this matters

- Production behavior is now easier to reason about because there is a single supported debugging surface again.
- The runtime has a little more breathing room for heavier scans without changing how the app works day to day.

## No action needed

You do not need to change your configuration or workflow after this release.
