# Career Jump API v1.1.1

**Released:** 2026-04-27

## Summary

Patch release focused on production readiness for auth recovery, shared scan reuse, and Workday registry-backed company scanning.

## Changes

### Auth and account recovery

- adds API-backed password reset storage and email flow
- supports both the main user pool and the admin user pool through the same reset experience

### Shared scan reuse

- stores raw fetched company jobs centrally so multiple users can reuse the same recent scan data
- reuses raw scan results for up to four hours before fetching again
- keeps per-user available job matching and inventory diffing on top of shared raw data

### Workday and registry-backed company handling

- adds typed Workday scan outcomes and persisted scan-state tracking
- introduces Cloudflare Worker egress for Workday Layer 1
- prefers canonical registry board URLs for discovered companies instead of depending on fragile sample posting URLs
- promotes validated custom companies into the shared registry so later users can add them directly

## Verification

- `npm run check:api`
- `npm run test:api`
- `npm run aws:build`

## Deployment Notes

- Deploy with `AWS_PROFILE=career-jump-prod-deployer AWS_REGION=us-east-1 npm run aws:deploy`
- Sync the frontend with `AWS_PROFILE=career-jump-prod-deployer AWS_REGION=us-east-1 npm run aws:sync-frontend`
