# Career Jump AWS v4.0.1

**Released:** 2026-04-25

## Summary

Patch release. Fixes a Lambda bundling bug in v4.0.0 that would have caused all `/api/registry/*` endpoints to return 500 in production.

## Bug Fix

### Registry data not bundled into Lambda artifact

**Symptom (would have been):** First call to `/api/registry/meta`, `/api/registry/companies`, `/api/registry/scrape`, or `/api/registry/scrape/ats` throws `ENOENT: no such file or directory, open '.../data/seed_registry.json'`.

**Root cause:** [src/storage/registry-cache.ts](src/storage/registry-cache.ts) used `readFile()` with a path resolved from `import.meta.url`. SAM's esbuild bundler emits a single `api.js` per function and does not copy the `data/` folder. At runtime the relative path resolves to a location that doesn't exist in the deployed bundle.

**Fix:** One-file change in [src/storage/registry-cache.ts](src/storage/registry-cache.ts):

- Removed runtime `readFile` + `__dirname` path resolution as the default code path
- Added static import: `import bundledRegistry from "../../data/seed_registry.json"`
- `loadRegistryCache()` uses the inlined JSON by default; `opts.path` escape hatch retained for tests/overrides

esbuild now inlines the registry payload directly into each function bundle.

**Caught by:** SAM build artifact inspection during v4.0.0 pre-deploy verification — never shipped to production.
