# Career Jump AWS v2.2.0

Release date: 2026-04-22

## Summary

This release brings the AWS POC up to the Cloudflare production v2.2.0 feature set while keeping the AWS-specific Cognito, CloudFront, S3, Lambda Function URL, DynamoDB, and GitLab release pipeline architecture.

## Product Changes

- Added role notes to Available Jobs, Applied Jobs, and Action Plan.
- Notes use a comment-thread modal: click the notes icon, add one or more comments with Send, then Save to persist.
- Notes follow the job lifecycle: applying a role carries notes into Applied Jobs, moving into Interview carries notes into Action Plan, and discarding/removing an available role removes its notes.
- Added a visible notes state so roles with notes are distinguishable at a glance.
- Added discard buttons to Available Jobs without writing discard events into `/logs`.
- Grouped Applied Jobs by stage: Applied, Interview, Negotiations, Offered, and Rejected.
- Added per-company scan pause toggles plus a global all-active/all-paused toggle in Configuration.
- Updated scan progress counts so paused companies are excluded from the progress denominator.

## Storage And Scan Behavior

- Fetched ATS jobs are still counted before filters, but only matched inventory plus applied-job state are persisted.
- Paused-company inventory is preserved across scans.
- Active companies are refreshed normally: new and updated jobs are appended/updated, discarded filtered jobs are not stored, and jobs missing from a fresh active-company scan are removed.
- Manual discard keys suppress discarded available jobs from storage and UI on future saves.
- Per-company scan logs remain one summarized row per company with fetched, matched, excluded, updated, discarded, and progress details.
- Final AWS fanout completion writes a single `run_completed` summary.

## AWS Release Notes

- Version files updated to `2.2.0`.
- Static frontend cache-busting updated to `v=2.2.0`.
- OpenAPI version updated to `2.2.0`.
- The AWS Cognito browser auth flow is unchanged.
- CloudFront is retained because it provides HTTPS for Cognito callbacks, private S3 origin access, SPA routing, compression, and cache invalidation with negligible expected cost for a single-user POC.
- API Lambda reserved concurrency increased from 5 to 20 and browser API calls retry transient HTTP 429 responses to avoid refresh/run-polling errors.
