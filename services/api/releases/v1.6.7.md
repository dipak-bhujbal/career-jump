# Career Jump v1.6.7

Career Jump now has a deployable AWS proof-of-concept path that keeps the Cloudflare production app separate from the GitLab AWS lane.

## What's improved

- Added a SAM/CloudFormation stack for the AWS POC.
- Added a Cognito-protected Lambda Function URL API with no API Gateway dependency.
- Added a CloudFront HTTPS frontend so Cognito can safely redirect back to the app.
- Added browser-side Cognito hosted login and API token forwarding.
- Added a frontend sync script that publishes static assets plus generated AWS runtime config.
- Added a DynamoDB-backed runtime state adapter for the existing app storage model.
- Added parallel company scan fanout with one generic scan Lambda invocation per company.
- Added a finalization Lambda that merges company results and updates inventory after all scans complete.
- Added one-day CloudWatch log retention and disabled scheduled runs by default for first-deploy safety.

## Why this matters

The AWS POC can now be deployed as repeatable infrastructure while keeping the monthly-cost target below 5 USD and avoiding always-on services such as NAT Gateway, RDS, EC2, ALB, and API Gateway.

## Reliability notes

- EventBridge schedules are present but disabled until secrets and runtime data are ready.
- Apps Script webhook settings are deploy parameters and are intentionally blank for the first deployment.
- The first AWS deployment is intended to validate infrastructure, authentication, and manual run orchestration before scheduled scans are enabled.
- The S3 website endpoint remains available for low-level static hosting checks, but the usable app entrypoint is the CloudFront HTTPS URL.
