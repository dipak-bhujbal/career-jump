# Career Jump API 2.0.0

## Overview

Version 2.0.0 establishes a clean release baseline for the API after consolidation cleanup and introduces the next operational foundation for shared scan orchestration, registry safety, and board-URL-based configuration handling.

## Highlights

- Added registry scheduling and worker foundations for broader scan orchestration.
- Improved configuration normalization so canonical board URLs remain the source of truth.
- Strengthened shared raw-scan and inventory handling to support reuse and registry promotion behavior.
- Added integration and end-to-end coverage for registry routability, scheduler state, worker flows, and run lifecycle behavior.

## Operational Improvements

- Registry-backed companies are normalized against canonical board URLs rather than relying on individual sample postings.
- New registry scan worker and dashboard code paths establish the operational scaffolding for controlled background scanning and visibility.
- Release artifacts were reset so release documentation and Git tags can stay synchronized from this point forward.

## Risk and Compatibility Notes

- This release includes infrastructure template updates and new Lambda entry points; SAM validation and build checks were completed before deployment.
- The release intentionally favors explicit board-driven configuration over legacy sample-URL dependency.

## Validation

- TypeScript API checks passed.
- Full API test suite passed.
- SAM build passed for all packaged Lambda functions.
- SAM template lint validation passed prior to deployment.
