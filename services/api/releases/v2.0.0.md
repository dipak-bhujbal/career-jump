# Career Jump v2.0.0

Career Jump AWS is now packaged as the v2.0 release line for the GitLab-based AWS POC.

## What's improved

- Moved the AWS POC into a documented v2.0 release shape with CloudFront, S3, Cognito, Lambda Function URLs, DynamoDB, SAM, and a protected GitLab release pipeline.
- Added the in-page scan progress bar for parallel AWS runs so the dashboard shows `x/y companies fetched` instead of noisy per-company progress toasts.
- Compact `/logs.html` by default so repeated scan internals collapse into run-level summaries while raw troubleshooting rows remain available with `compact=false`.
- Updated the architecture pack with AWS service topology, runtime flow diagrams, state diagrams, and AWS Architecture Icon service references.
- Added a single `release-prod` GitLab job for intentional production deploys from release branches or release tags.

## Why this matters

The AWS POC now has a release process and operating model that match how it is actually deployed: GitLab for version control, SAM/CloudFormation for infrastructure, CloudFront for the app, Cognito for owner login, Lambda for parallel scan execution, and DynamoDB for state.

## Reliability notes

- The release keeps the low-cost serverless boundary: no API Gateway, NAT Gateway, RDS, EC2, ALB, Fargate, OpenSearch, or Step Functions.
- CloudWatch log groups remain one-day retention; application logs stay in DynamoDB with compact operator views.
- Production deploys should be run through the protected GitLab `release-prod` job after tagging `v2.0.0`.
