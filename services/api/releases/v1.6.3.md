Career Jump 1.6.3 focuses on scan reliability and operational clarity.

This release hardens the Workday scan path so one problematic company is much less likely to stall the whole run, adds better recovery when a scheduled scan dies mid-company, and introduces a proper architecture pack in the repository so the production system is easier to understand and maintain.

## What’s improved

- Workday scans now have stronger timeout protection at the page level as well as the company level.
- The app now records active-run heartbeats with company, stage, and page progress so stuck scans are easier to detect and recover.
- Stale active locks can now be recovered automatically when a new run proves the old run is no longer making progress.
- Timeout-specific logs now explain where a Workday scan stopped, including the last known fetch progress.
- A single Workday company timeout now fails that company and allows the overall run to continue instead of killing the whole scan.

## Architecture and operations

- Added a GitHub-rendered architecture pack under `docs/architecture/` with:
  - infrastructure diagrams
  - API and scan flow diagrams
  - runtime state diagrams
- Updated the production runbook so every future production release explicitly reviews and updates those diagrams when needed.

## Reliability notes

- This release is designed to reduce repeated “stuck on one Workday company” incidents, especially during scheduled scans.
- The new logs should make repeated offenders much easier to identify and tune in future releases.
