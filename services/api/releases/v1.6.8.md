# Career Jump v1.6.8

Career Jump AWS POC now fixes browser API fetches from the CloudFront app.

## What's improved

- Removed duplicate CORS headers from Lambda Function URL responses so browser requests to the AWS API are accepted.
- Kept app-owned CORS headers on every API response, including errors and preflight requests.
- Stored the short-lived Cognito login state in local storage as well as session storage so Docs and Logs opened in a new tab can use the same authenticated browser session.
- Bumped frontend asset URLs to refresh the deployed app, docs, logs, and AWS runtime config.

## Why this matters

Configuration loading, saving companies, app refresh, Swagger/OpenAPI loading, and logs all depend on browser fetches to the Lambda API. This release removes the network-layer failure that surfaced as `Failed to fetch`.
