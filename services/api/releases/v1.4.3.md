# Career Jump 1.4.3

This release improves notification consistency and trims noisy scan logs.

## What’s improved

- Email notifications now use the same stored new and updated job markers that power the inventory page.
- Updated jobs in email are now separated into their own section instead of being folded into the new-jobs flow.
- Updated-job emails can include field-level change details so it is easier to see what changed.
- Removed the repetitive `newJobsReason` explanation from company scan completion logs.

## Why this matters

- The inventory page and email notifications should now stay aligned much more closely for each completed run.
- Updated jobs are easier to review because they are clearly separated from brand-new openings.
- Operational logs stay more compact and easier to scan.

## No action needed

Your existing workflow does not change with this release.
