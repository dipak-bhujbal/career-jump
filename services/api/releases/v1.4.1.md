# Career Jump 1.4.1

This is a small reliability release focused on cleaner scheduling and lighter operational logging.

## What's improved

- Production scheduling now uses direct Cloudflare cron expressions instead of hourly wake-ups with code-based gating.
- Company scan logging is more efficient and easier to read.
- Each completed company scan now keeps one completion log with the useful summary details in one place.

## Why this matters

- Scheduled runs are simpler to reason about in Cloudflare.
- Fewer unnecessary cron invocations should reduce noise and make failures easier to spot.
- Logs stay useful without repeating the same company summary in multiple entries.

## Reliability notes

- No user action is required.
- Existing tracked jobs, configuration, and history remain intact.
- This release does not change the core job-matching workflow.
