# Career Jump v2.0.2

Career Jump v2.0.2 fixes the GitLab SAM build failure and tightens the compact `/logs.html` run timeline.

## What Changed

- Moved `esbuild` into production dependencies so GitLab's clean SAM build container can find the esbuild binary.
- Compact `/logs.html` now emits only the scan timeline:
  - one `scan_started` row
  - one `company_scan_summary` row per completed, failed, skipped, or aborted company
  - one `inventory_final_built` row with inventory and email outcome
- Company summary details include fetched, matched, new, updated, updated-job diffs, excluded-title, excluded-geography, discarded count, duplicate count, suppressed-seen count, duration, and progress.
- Final inventory details include `totalMatched`, `totalNewMatches`, `totalUpdatedMatches`, `totalFetched`, `emailStatus`, `emailSkipReason`, and `emailError`.

## Operator Notes

- Deploy through the GitLab release pipeline using tag `v2.0.2`.
- The previous failed GitLab build was caused by `sam build` not finding `esbuild` in the CI container.
