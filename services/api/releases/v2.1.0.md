# Career Jump API 2.1.0

## Overview

Version 2.1.0 strengthens the operational API surface for Career Jump with administrative billing foundations, plan-configuration persistence, and restored structured note handling for applied jobs. This release focuses on keeping production workflows aligned with the intended product model while preserving deployment safety in the current AWS environment.

## Highlights

- Added backend support for plan configuration, billing-oriented administration, and related route coverage.
- Restored applied-job note records with explicit add, update, and delete APIs.
- Improved applied-job persistence so legacy single-note data is lifted into structured note records instead of being lost.
- Expanded integration and smoke coverage around run lifecycle, inventory-plan flows, registry operations, plan limits, and notes behavior.

## Operational Impact

- Applied-job notes are now stored and returned as structured records while retaining backward compatibility with older note payloads.
- API responses for applied jobs and action-plan flows now preserve richer workflow state required by the web experience.
- Direct production code publishing remains possible for targeted API fixes even while broader infrastructure permissions are still being tightened.

## Risk and Compatibility Notes

- This release includes code-path expansion across routes, storage normalization, inventory handling, billing, and admin configuration.
- Legacy flattened notes remain readable, but newly saved applied-job notes now follow the structured record model expected by the UI.

## Validation

- TypeScript API checks passed.
- Focused smoke coverage for applied-job note CRUD passed.
- SAM build completed successfully for the packaged API artifacts.
