# Career Jump 1.5.1

This release tightens how Career Jump is shipped so patch releases come from a dedicated release branch instead of being cut ad hoc from the main development line.

## What’s improved

- Standardized production releases around `release/<major>.<minor>` branches, starting with the `1.5.x` line.
- Updated the release workflow so published GitHub Releases show the actual production source branch in support metadata.
- Expanded repository documentation so the full release path is explicit from local validation through production deploy, tagging, and merge-back.

## Why this matters

- Production patches are easier to stabilize because the active release line is isolated from next-version development.
- Release notes and deployment metadata now better reflect how the release was actually produced.
- The documented process is now consistent with the Git flow expected for future patch releases.

## No action needed

This release does not change app behavior or your production workflow inside Career Jump itself.
