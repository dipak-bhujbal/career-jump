Career Jump 1.6.5 makes updated-job detection more trustworthy.

This patch fixes a case where the app could mark a job as updated only because it matched a different keyword than it did in a previous run. Keyword match metadata is useful for filtering and display, but it should not by itself count as a meaningful job change.

What’s improved

- A job is no longer treated as updated when the only change is the matched keyword set.
- Updated-job badges, counts, and notifications now focus on real changes to the job itself, such as title, location, URL, or geography signals.
- This reduces false-positive “Updated” states and keeps the available jobs view closer to what users actually expect.

Operational notes

- No architecture or API-flow changes were required for this patch.
- The existing 1.6.4 scan-core rollback remains in place unchanged.
