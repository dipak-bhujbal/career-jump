# Career Jump v1.0.0

**Released:** 2026-04-27

## Summary

Career Jump v1.0.0 establishes the first production-ready AWS release line for the unified SaaS platform. This release consolidates the React frontend and serverless backend into a single deployable monorepo, introduces the multi-tenant account foundation, adds dedicated admin and support operating surfaces, and hardens the edge boundary for a US-only launch.

This release is intended to serve as the operational baseline for future releases. It prioritizes secure deployment, tenant-aware data separation, supportability, and disciplined release management over additional growth features.

## Highlights

- **Fresh AWS production baseline** with SAM-managed Lambda, DynamoDB, S3, CloudFront, Cognito, and SES.
- **Multi-tenant account foundation** with user, billing, support, events, jobs, and raw-scan tables separated for the SaaS model.
- **Separate admin authentication plane** using a dedicated Cognito user pool for administrative access.
- **Admin and support product surfaces** for customer support operations, tenant-level review, feature-flag management, and announcement publishing.
- **Break-glass log access controls** so administrators can only access logs for a specific customer with explicit justification and audit capture.
- **US-only edge enforcement** through CloudFront georestriction plus backend country and origin verification.
- **Session enforcement groundwork** for concurrent-session limits, anomaly logging, and account-sharing deterrence.
- **SES-based notification delivery** replacing the older webhook-driven email path.

## Platform Changes

### Multi-tenant backend foundation

- Added dedicated DynamoDB tables for `users`, `billing`, `support`, `events`, `jobs`, and `raw-scans`.
- Preserved existing state and registry storage so deployment and migration can happen incrementally.
- Added tenant-aware request resolution and storage scoping to prepare the application for true multi-user production operation.

### Authentication and access boundaries

- Replaced the prior single-user email gate with Cognito-backed user authentication.
- Added a separate admin Cognito user pool and runtime configuration path for administrative sign-in.
- Added production-only request hardening so authenticated API traffic must pass through the approved edge path and carry a valid session identifier.

### Admin and support operations

- Added user-facing support ticket APIs and frontend support screens.
- Added admin summary, user management, support queue, feature-flag, and announcement APIs and UI routes.
- Restricted operational logs to administrators and converted log access into a break-glass workflow that requires a customer target and justification.

### Security, compliance, and launch controls

- Added CloudFront US geofencing for the frontend distribution.
- Added backend enforcement for `cloudfront-viewer-country` and the private edge-origin verification header.
- Added session tracking records, concurrent-session eviction, and anomaly event capture for suspicious device and IP patterns.

### Email and operational delivery

- Swapped the prior Apps Script notification path for SES-based email delivery.
- Added runtime configuration needed for admin auth, frontend API routing through CloudFront, and same-origin application traffic.

## User Experience Changes

- Added a dedicated `/support` experience for end-user ticket submission and history.
- Added `/admin`, `/admin/users`, `/admin/support`, and `/admin/flags` administrative routes.
- Removed logs from the standard user navigation.
- Updated the logs experience so it is available only to administrators in a customer-scoped break-glass flow.

- `services/api/src/aws/auth.ts` — Cognito auth verification plus edge and session enforcement

- `services/api/src/storage/accounts.ts` — account, billing, support, and session persistence
- `services/api/src/routes.ts` — admin, support, `/api/me`, and break-glass logs APIs
- `services/api/src/services/email.ts` — SES email delivery path

## Verification

- `npm run validate`
- `sam validate --lint --region us-east-1`
- `npm run aws:build`

## Deployment Notes

- Deploy this release from the `release/1.0.0` branch using the dedicated deployer IAM profile.
- Provide a real `OriginVerifyHeaderValue` at deploy time; do not store it in source control.
- Verify SES identities in `us-east-1` before expecting production email delivery.
- Restore the preserved registry backup after the stack is created and the registry table exists.

## Rollback

- Roll back by redeploying the prior known-good tag or stack template and re-syncing the frontend assets.
- No destructive data migration is required for this release baseline, so infrastructure rollback is primarily a redeploy decision rather than a data-rewrite event.
