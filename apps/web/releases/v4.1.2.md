# v4.1.2

Patch release for the registry API deployment blocker found during the
`v4.1.1` AWS rollout.

## Highlights

- Removes `OPTIONS` from the Lambda Function URL CORS `AllowMethods` list.
  Lambda Function URLs handle preflight separately and only accept actual HTTP
  methods in this property.
- Keeps the `v4.1.1` registry routing and configuration UI parity changes
  unchanged.

## Validation

- `npm run lint`
- `npm run build`
- `git diff --check`
- `sam validate --lint --template-file infra/registry-api.yaml`
- CloudFormation template validation for `infra/registry-api.yaml`
- CloudFormation template validation for `infra/full-stack.yaml`
