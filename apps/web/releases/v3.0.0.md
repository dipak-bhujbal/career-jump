# Career Jump Web 3.0.0

## Overview

Version 3.0.0 expands the Career Jump web application from a single-user scanning surface into a more complete operating console. This release introduces a fuller admin workspace, live announcement management, embedded admin documentation, company-specific applied-job views, and broader billing, support, and route-level workflow polish.

## Highlights

- Added a richer `Admin Workspace` landing page with direct access to users, support, feature flags, analytics, plan configuration, Stripe configuration, live announcements, and embedded API docs.
- Added `Admin Announcements`, allowing operators to create, target, schedule, pause, and remove user-facing product notices without editing storage directly.
- Added `Admin Docs`, embedding the live Swagger/OpenAPI surface inside the product so admins can inspect the shipped contract without leaving the app.
- Added a dedicated company-level applied-jobs experience so users can review one company pipeline at a time as either a board or grouped list.
- Expanded applied-job workflows, sidebar actions, support queries, and admin route coverage across the product shell.

## User Experience Improvements

- Admin navigation is now organized as an operational control room rather than a collection of hidden direct routes.
- User-facing announcements can be targeted by plan or tenant, enabling narrower rollouts and operational notices.
- Applied-jobs tracking now supports a company-specific page for faster review of interview progression, negotiation stages, and offer outcomes within a single employer pipeline.
- Billing and upgrade surfaces received iterative UX polish so plan controls and subscription flows remain consistent with the growing admin surface.
- Route registration and app-level shell updates now keep new admin and applied-company flows visible and easier to discover.

## Operational Impact

- Admins can now manage live messaging and inspect API documentation directly inside the deployed application.
- Product and support teams gain clearer visibility into analytics, support activity, pricing policy, and rollout controls from one workspace.
- Company pipeline review is no longer limited to the global applied-jobs board, reducing navigation overhead for focused follow-up work.

## Reliability and Quality

- Added route-level test coverage for admin announcements, admin docs, and company applied-job views.
- Preserved compatibility with the existing product shell while broadening the admin navigation model and route tree.
- Continued surfacing the packaged application version inside the UI so deployed-version verification remains visible during support and QA workflows.

## Validation

- Production web build completed successfully.
- API type checks and API test suites passed before deployment.
- The 3.0.0 production stack deploy completed successfully after the deployer policy was expanded to allow the new queue-backed infrastructure.
