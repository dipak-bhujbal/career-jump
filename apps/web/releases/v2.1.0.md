# Career Jump Web 2.1.0

## Overview

Version 2.1.0 advances the product from baseline stabilization into operational administration and day-to-day workflow polish. This release expands billing and plan-management surfaces, restores threaded applied-job notes, and improves version transparency so product, support, and QA can confirm what is live without leaving the application.

## Highlights

- Added new admin surfaces for commercial configuration, including Stripe and plan-management controls.
- Restored record-style applied-job notes with add, edit, and delete behavior in the job details workflow.
- Improved product version visibility by aligning both Profile and Settings with the packaged release version.
- Expanded frontend coverage around new admin route behavior and workflow regressions.

## User Impact

- Applied jobs once again support note threads instead of a single flattened note field, improving interview tracking and follow-up logging.
- Administrators gain clearer in-product controls for subscription and plan configuration.
- The deployed release version is visible from the app itself, reducing ambiguity during support and validation.

## Reliability and Quality

- Added regression coverage to protect the applied-job notes contract and route behavior.
- Refined shared query invalidation and UI state flows across applied jobs, billing, admin views, and support interactions.
- Preserved compatibility with the current production API contract while restoring richer note interactions for applied jobs.

## Validation

- Production web build completed successfully.
- API type checks and focused regression tests passed for the applied-job notes workflow before deployment.
