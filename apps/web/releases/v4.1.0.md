# v4.1.0

Minor release for the separated registry data path and supporting AWS
foundation updates.

## Highlights

- Adds the dedicated `career-jump-web-poc-registry` DynamoDB table to the
  React-owned foundation stack.
- Adds a registry seed script that defaults to the dedicated registry table.
- Adds registry API infrastructure for DynamoDB-backed registry reads.
- Updates frontend CloudFront infrastructure to optionally route
  `/api/registry/*` to a registry API origin when configured.
- Keeps company configuration display behavior aligned with persisted registry
  metadata.
- Removes first-visit command-palette auto-open behavior.

## Validation

- `npm run lint`
- `npm run build`
- `git diff --check`
- CloudFormation template validation for changed infra templates
