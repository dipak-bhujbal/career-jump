# v4.0.0

Major release for the latest local company-management and cache-refresh work.

## Highlights

- Uses backend registry search directly instead of bundled registry search
  fallback, while retaining bundled registry metadata for empty-meta handling.
- Improves company picker behavior around existing tracked companies and search
  state.
- Treats registry-origin companies consistently even when the backend persists
  registry metadata but not the `isRegistry` boolean.
- Drops stale applied-job rows with missing job payloads from applied filters.
- Clears cached jobs, applied jobs, action plan, dashboard, and logs immediately
  after data reset.
