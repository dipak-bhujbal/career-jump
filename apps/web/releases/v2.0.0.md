# Career Jump Web 2.0.0

## Overview

Version 2.0.0 establishes the first disciplined release baseline for the web application after repository consolidation cleanup. This release improves configuration safety, adds clearer product version visibility, and aligns the user-facing company onboarding flow with canonical job board URLs instead of individual sample postings.

## Highlights

- Canonical board URL handling now drives registry-backed and custom company configuration flows.
- Profile now displays the shipped release version to make deployed-version verification immediate for support, QA, and product review.
- Legacy release-note carryover files were retired so the release inventory can restart from a clean baseline.

## User Impact

- Users adding companies from the registry now work from canonical board URLs rather than fragile single-job posting URLs.
- Custom company creation is better aligned with real scan behavior and future registry promotion rules.
- Support and product reviews can confirm the active frontend release directly from the Profile page.

## Reliability and Governance

- Release metadata is now injected at build time so the displayed version matches the packaged web artifact.
- Consolidated release history artifacts were removed to restore one authoritative release lineage.

## Validation

- Production web build completed successfully.
- Integrated API validation and test gates passed for the release candidate before deployment.
