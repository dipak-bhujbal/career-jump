# v2.0.0

Authentication, account flows, and production-readiness release for
`career-jump-web`.

## Highlights

- Adds a real Cognito-backed auth layer with login, signup, email verification,
  forgot-password, and protected-route handling while retaining mock mode for
  local development when Cognito env vars are absent.
- Expands profile and settings surfaces for account management, privacy flows,
  and user-facing notification preferences.
- Adds multi-tenancy, auth, and email architecture documentation plus an agent
  handoff document that captures the remaining backend scoping work.
- Adds deployable infrastructure templates for Cognito, SES, the notification
  Lambda, and a full-stack orchestration path alongside the isolated frontend
  hosting stack.

## Included work

- New auth routes:
  - `/login`
  - `/signup`
  - `/verify-email`
  - `/forgot-password`
  - `/privacy`
- New auth UI and context wiring under `src/components/auth/` and
  `src/features/auth/`
- Expanded profile/settings flows
- New infrastructure docs and templates under `infra/`

## Validation

- `npm run build`

## Deploy shape

- Frontend stack: `career-jump-web-poc`
- Static bucket: `cj-web-static-poc-<account-id>`
- CloudFront distribution: isolated from the older `career-jump` frontend

## Follow-up

- The frontend is ready for real Cognito configuration, but multi-tenant data
  isolation still depends on implementing the backend scoping stubs in
  `career-jump-aws`.
