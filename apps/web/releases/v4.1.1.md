# v4.1.1

Patch release for production registry routing and configuration UI parity.

## Highlights

- Adds `registryBaseUrl` runtime configuration so registry reads can use the
  dedicated `career-jump-web` registry Lambda while the rest of the app stays
  on the existing API during the backend transition.
- Routes Add Company, registry search, registry metadata, and company hover-card
  lookups through the registry-specific API client.
- Adds single-company lookup support to the registry Lambda for hover-card
  details.
- Backfills registry-backed company display from persisted source/sample URL
  signals so production rows render like local auto-tracked rows instead of
  editable custom rows.
- Keeps registry calls backward-compatible by falling back to the main
  `apiBaseUrl` when `registryBaseUrl` is not configured.
