# v4.1.4

Patch release for registry search CORS behavior.

## Highlights

- Removes duplicate CORS headers from registry Lambda responses.
- Leaves CORS ownership with the Lambda Function URL configuration so browser
  registry searches receive a single `Access-Control-Allow-Origin` header.
- Keeps Cognito bearer-token validation and the dedicated registry table wiring
  from the prior patch releases.

- `npm run lint`
- `npm run build`
- `git diff --check`
- `sam validate --lint --template-file infra/registry-api.yaml`
- CloudFormation template validation for `infra/registry-api.yaml`
