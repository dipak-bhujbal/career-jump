# v4.1.3

Patch release for the registry Lambda Function URL public invoke policy.

## Highlights

- Adds the required `lambda:InvokeFunction` resource policy statement for the
  registry Lambda Function URL.
- Restricts that invoke permission with `InvokedViaFunctionUrl: true` so it
  only applies to Function URL traffic.
- Keeps the existing `lambda:InvokeFunctionUrl` permission and Cognito JWT
  validation inside the registry Lambda.
