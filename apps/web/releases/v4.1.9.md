# v4.1.9

Patch release for Add Company picker behavior and deployment safety.

## Highlights

- Changes Add Company to use one flat result list instead of grouped starter
  tier sections.
- Makes `All`, `Tier 1`, `Tier 2`, and `Tier 3` act as true filters: selecting
  a tier shows only that tier, and searching stays scoped to the selected tier.
- Broadens registry search response normalization so older/newer response
  aliases do not blank the picker.
- Restores production runtime config and updates the deploy process to avoid
  overwriting `/aws-config.js` with local placeholders.
