# Release Notes — v3.0.0

**Release date:** 2026-04-28  
**Branch:** main  
**Previous version:** v2.1.0

---

## Summary

Version 3.0.0 is a platform expansion release. It extends the admin workspace into a more complete operations console, introduces live announcement and embedded documentation workflows, adds dedicated company-level applied-job views, and ships the first queue-backed registry scan infrastructure in production.

---

## What’s New

### 1. Expanded Admin Workspace

- The admin landing page now acts as an operational control room rather than a simple route directory.
- Admin operators now have direct in-product access to:
  - Users
  - Support queue
  - Logs
  - Feature flags
  - Plan configuration
  - Stripe configuration
  - Announcements
  - Embedded API docs
  - Analytics

### 2. Live Announcements Management

- Added a dedicated announcements workflow inside admin.
- Operators can create and manage notices with:
  - title
  - body
  - severity
  - active or paused state
  - dismissible or persistent behavior
  - active-from and active-to windows
  - target plans
  - optional tenant targeting
- This allows live operational messaging without direct storage edits or redeploys.

### 3. Embedded Admin Documentation

- Added an admin docs route that embeds the shipped Swagger/OpenAPI surface directly inside the product.
- Admins can verify the live API contract, payload shapes, and admin-only routes without leaving the application or opening separate tools.

### 4. Company-Level Applied Pipeline View

- Added a dedicated company-specific applied-jobs view.
- Users can open a single company pipeline and switch between:
  - board view
  - grouped list view
- The company route makes it easier to review one employer pipeline without losing context inside the larger applied-jobs board.

### 5. Queue-Backed Registry Scan Infrastructure

- Production now includes dedicated SQS-backed scan queues for:
  - Workday
  - Enterprise ATS providers
  - Public API adapters
- Added a registry scheduler that selects due companies and routes them into the appropriate queue based on adapter type.
- Added queue-connected worker infrastructure to process those scheduled scan messages.
- Added supporting operational resources:
  - CloudWatch alarms
  - log metric filters
  - log groups
  - scheduler resources
  - registry operations dashboard function

This is the most significant backend platform change in the release.

### 6. Broader Admin and API Surface

- Expanded admin-facing routes and storage support across:
  - analytics
  - billing
  - support
  - announcements
  - company pipeline flows
- Expanded OpenAPI coverage so the shipped documentation better matches the live product behavior.

---

## User Impact

- Admins now have a much stronger in-product control surface for operating the system.
- Users and support workflows benefit from clearer operational messaging and a more focused company-level pipeline experience.
- Product and QA teams gain better deployed-contract visibility through embedded docs and broader analytics/admin navigation.

---

## Infrastructure and Deployment Notes

- The full production stack for `career-jump-prod` was updated successfully in `us-east-1`.
- New production resources created in this release include:
  - `career-jump-prod-queue-workday`
  - `career-jump-prod-queue-enterprise`
  - `career-jump-prod-queue-public-api`
  - registry scheduler resources
  - registry worker resources
  - registry ops dashboard resources
- During rollout, the deployer IAM user initially lacked `sqs:CreateQueue`, which blocked the first CloudFormation attempt.
- After the deployer policy was expanded, the full production deployment completed successfully.

## Breaking Changes

None declared at the application contract level.
